# Task Cockpit · 你的个人任务驾驶舱

> 把项目管理这件事，整个交给 agent。你只管创造。

## 这是给谁的

如果你是同时对接多个项目的打工人——尤其是互联网行业——你一定熟悉这种感觉：

一边要全神贯注做产出，一边脑子里还有一堆后台进程在跑：*这个项目我做到哪了？还剩什么没做？接下来该干哪个？哪件事最急？那件事是不是在等别人？* 这些问题不停打断你，真正消耗你的不是任务本身，而是**持续追踪任务状态的心智负担**。

到了季度述职、晋升答辩、写周报的时候，又是另一种痛：明明忙了三个月，可一旦要"总结你做成了什么"，却怎么也想不起来——那些真正有价值的成果，早就淹没在日常琐碎里了。

**Task Cockpit 就是来接管这两件事的。**

## 它怎么改变你的一天

你不用学任何新的操作界面，不用记快捷键，不用手动拖卡片。你只需要**用大白话跟 agent 说话**：

- 「我接了个 App 改版，要改登录页、加埋点、还要灰度发布；另外年报下周三交初稿」
  → agent 自动帮你拆成项目和任务，标好优先级，放进看板等你过目。觉得哪里不对，直接说「灰度那条先不急」「年报拆成核对和撰写两步」，它就改。

- 「登录页改稿做完了」
  → agent 不只是打个勾。它会顺手帮你记下这件事**做成了什么**，沉淀成一句随时能用的成就陈述。

- 「我现在该干啥？」
  → agent 看一眼全局，告诉你最该动的是哪个、哪个在等别人可以先放着、哪个有空就能推。

- 「帮我总结这季度在 App 改版上的成果，要用在述职」
  → agent 从你日积月累的成就库里，按述职 / 周报 / 复盘的不同写法，给你组织好材料。**只用你真实记录过的东西，绝不编造。**

与此同时，一个**只读看板**始终开在浏览器里，顶部是 agent 算出的"今日聚焦"，下面是按项目铺开的全貌。你想看一眼进度，瞟一眼就好，不用动手维护。

## 为什么是"完成即沉淀"

这是 Task Cockpit 最不一样的地方。

大多数待办工具，任务做完就消失了。但你的**成长和价值**恰恰藏在这些做完的事里。Task Cockpit 把每一个完成的任务，连同它的成果、甚至你的复盘反思，都永久留存下来——

- 对**你个人**：再也不会在述职前抓瞎。你做过的每一件有价值的事都在，越积越厚，你会越来越清楚自己究竟创造了什么。
- 对**团队组织**：那些复盘和教训，是可以沉淀、可以传承的财富。

成就不该是答辩前熬夜拼凑的回忆，而该是平时一点一滴、自动攒下来的资产。

## 它由什么组成

三个部分，各司其职，数据与代码分离：

| 文件 | 职责 |
| --- | --- |
| `cockpit.py` | 数据层 + 命令行（全部业务逻辑，纯 Python 标准库，零依赖） |
| `server.py` | 本地看板服务（固定端口 7842，仅监听 127.0.0.1，不闲置退出） |
| `dashboard.html` | 只读看板（每 2 秒自动刷新，离线可用） |
| `SKILL.md` | 教 agent 如何识别你的意图、如何驱动这套工具 |

你的数据存在本机 `~/.task-cockpit/`，首次使用自动创建，完全属于你自己：

- `projects.json` —— 你的项目
- `tasks.json` —— 当前任务快照
- `achievements.jsonl` —— 成就库（只增不减，你的 CV 原料）
- `cv-exports/` —— 导出的述职 / 周报材料

## 安装

需要 Python 3（标准库即可，**无任何第三方依赖**）。把仓库克隆下来，软链接成 Claude Code 的 skill：

```bash
git clone https://github.com/CookieJobs/task-cockpit.git
ln -s "$(pwd)/task-cockpit" ~/.claude/skills/task-cockpit
```

装好后，在 Claude Code 里用自然语言触发即可。看板地址：http://127.0.0.1:7842

## 给团队用

整套工具不写死任何个人路径，开箱即用。想分享给同事，把本仓库套进一个 Claude Code plugin、自建一个 git marketplace（直接托管在 GitHub 即可），同事运行：

```bash
/plugin marketplace add https://github.com/CookieJobs/task-cockpit.git
```

每个人各自一份、数据独立存在自己机器上，互不干扰。
